# 文档说明

## 1. 这份文档的作用

`docs/` 目录用于存放 `my_app` 的项目说明、开发规则和结构约定，避免后续重构时目录混乱、接口配置分散、样式文件到处放。

## 2. 当前文档列表

- `PROJECT.md`
  说明当前项目定位、目录结构、接口约定、开发规范和运行方式。

- `文档说明.md`
  说明 `docs/` 目录本身的作用，以及项目内的一些统一约定。

## 3. 当前项目核心约定

### 认证项目

当前项目只保留登录和注册功能，其他页面已经归档，不参与当前版本开发。

### 接口域名统一配置

后端 API 域名统一写在：

- `src/config/config.js`

不要在页面或 hooks 里写死请求地址。

### 样式文件目录统一

所有后续新增的独立样式文件，统一放在：

- `src/stypes/`

这是当前项目的明确约定，后面开发请统一遵守。

### 历史页面保留但不启用

归档页面放在：

- `src/views/archive/`

这些页面只做参考，不直接参与当前项目的页面路由。

## 4. 后续更新文档时建议遵循

- 项目结构变化时，优先更新 `docs/PROJECT.md`
- 新增目录约定时，同时更新 `docs/文档说明.md`
- 样式目录规范如果变更，要同步修改这两份文档
