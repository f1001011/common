Integration Api

1.

Process Diagram

a.

Flow diagram of opening game API.

Flow Details

1.

The player login to the merchant platform.

2.

Merchant s server will get an Auth token with credentials provided by us.

3.

Our server will return the merchant s Auth Token.

4.

Merchant server will request our server to get a list of our games with

their token.

5.

GGSoft API server will return the game list.

6.

Merchant server will show the list to the player.

7.

The player then chooses our games from the list.

GGSOFT© Copyright (GREAT GAMING SOFT)

--- PAGE 6 ---
6

8.

Merchants need to get a token for the player before entering the game.

9.

GGSoft API server will return the player s Auth Token.

10.

Merchant server will call our API to get the URL for the selected games.

11.

GGSoft API server will return the redirect link to our games.

12.

Merchant platform will redirect the player to our game platform.

13.

Our game platform will validate player authentication to our server.

14.

Our server will process and the player can enjoy our game.

b.

Flow diagram of seamless wallet API

Flow Details

1.

Players click the start bet button inside our game platform.

2.

Our game platform sends requests to our server to process the bet.

3.

Our server will call the Merchant server to validate the bet amount if the

player has enough balance or not.

4.

Merchant server will process it and return the player s balance after

Merchant server reduces the player s balance with the bet amount.

5.

Our server will process the spin.

6.

Our server will return the winning result of the player.

7.

Merchant server will process and save the winning result, then return the

Player s balance back to our server.

8.

Our server will return the result to the game platform.

GGSOFT© Copyright (GREAT GAMING SOFT)

7

c.

Flow diagram of transfer wallet API

Flow Details

1.

Players exchanging merchant credit to ours, this will be done on the

Merchant server and platform.

2.

Merchant will request credits transfer to us.

3.

We will return the transfer result back to the Merchant.

4.

Merchant server will process it and return the player s balance.

Notes:

Before calling the transfer API, players need to be registered on the GGSoft

system. This process can be done when requesting player Auth (step No.8 on

Opening Game diagram), by calling

memberLogin service.

2.

Authentication & Token Usage

There are 2 kinds of authorization tokens that are used in the GGSoft

system.

%Ï

First, merchant token, which is used for accessing list games,

requesting game / bet records and required for player login / get

session. This merchant token credential is provided by GGSoft for any

merchant as a partner. This token is received by calling service in

section 3.b.

%Ï

Second, player token, which is used to get balance, transfer wallet,

enter game and any in-game session. This token also registers the

player automatically to the GGSoft system when called, if

uname

input does not exist on our data. This token is

not

required

if the

player is not accessing GGSoft games/transfer process. Please check

section 3.c for requesting this token authentication.

Check the table matrix below that shows the services list requiring each

token. Both merchant and player tokens are used as Bearer Token in API Service.

a.

Signature Algorithm

We use an MD5-based hashing mechanism to ensure the integrity and

authenticity of the request content. The signature is generated using the

following format:

md5(key1=value1&key2=value2&...&keyN=valueN + secret key)

Merchants are expected to use this method to verify that the request

content has not been tampered with.

GGSOFT© Copyright (GREAT GAMING SOFT)

The first layer of key order must be sorted alphabetically, Note: the

inside award_order_ids does not need to be sorted.

%Ï

%Ï

hashing verification failed.

The signing key provided in this documentation is intended for use in

our staging environment. A new signing key will be issued upon

transition to the production environment.
