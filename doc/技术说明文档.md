## 二、项目规范

### 1. 控制器 (Controller) 规范
- 命名：`别名+Con`，如 `GoodsOrderCon`
- 必须继承 `BaseCon`
- 方法定义：大写开头，驼峰命名

### 2. 模型 (Model) 规范
- 命名：`别名+Model`，如 `CommonGoodsOrderModel`
- 必须继承 `BaseModel`
- 常量定义：状态等固定参数必须定义到模型中
  ```php
  const STATUS_NORMAL = 0;
  const STATUS_DELETE = -1;
  ```

### 3. 数据库查询规范
- 分页查询必须使用 `paginate()`，通过模型封装的 `PageList()` 方法
- 单表分页：`Model::PageList($map, $field, $page, $limit, $order)`
- 单表不分页：`Model::PageData($map, $field, $limit)`
- 单表一条：`Model::PageDataOne($map, $field)`

### 4. 接口返回规范
- 必须使用 `Show()` 函数返回数据
- 格式：`Show($code, $data, $msg)`
- 状态码定义在 `app/helper/Code.php`
- 消息定义在 `app/lang/` 目录下

### 5. 参数验证规范
```php
$postField = 'phone,pwd,captcha';
$post = $this->request->only(explode(',', $postField), 'post', null);

try {
    validate(\app\validate\LoginValidate::class)
        ->scene('login')
        ->check($post);
} catch (ValidateException $e) {
    return Show(ERROR, [], $e->getError());
}
```

### 6. 分页参数规范
- `page`: 当前页，默认 1
- `limit`: 每页数量，默认 20

### 7. 防并发锁规范
```php
$lockKey = 'action_lock_' . $userId . '_' . $goodsId;
if (Cache::get($lockKey)) {
    return Show(ERROR, [], 10016); // 操作频繁
}
Cache::set($lockKey, 1, 5);
// 业务逻辑...
Cache::delete($lockKey);
```

### 8. 事务处理规范
```php
Db::startTrans();
try {
    // 业务逻辑...
    Db::commit();
    return Show(SUCCESS, $data, 'success_code');
} catch (\Throwable $e) {
    Db::rollback();
    return Show(ERROR, [], 'error_code');
}
```

## 五、常用方法

### 用户余额操作
```php
// 增加余额
CommonUserModel::incMoney($userId, $amount);

// 减少余额（有锁）
CommonUserModel::decMoney($userId, $amount);

// 减少积分（有锁）
CommonUserModel::decIntegral($userId, $amount);
```

### 资金流水记录
```php
CommonPayMoneyLogModel::recordMoneyLog(
    $userId,           // 用户ID
    $type,             // 类型：TYPE_INCOME / TYPE_EXPEND
    $status,           // 状态：具体业务状态码
    $moneyType,        // 资金类型：MONEY_TYPE_BALANCE / MONEY_TYPE_INTEGRAL
    $money,            // 变动金额
    $moneyBefore,      // 变动前金额
    $moneyEnd,         // 变动后金额
    $remark            // 备注
);
```

### 收益领取相关
```php
// 获取订单待领取数量
CommonIncomeClaimLogModel::getPendingCount($orderId);

// 获取用户所有待领取数量
CommonIncomeClaimLogModel::getUserTotalPendingCount($userId);

// 获取最近即将过期的待领取记录
CommonIncomeClaimLogModel::getNearestPendingClaim($userId);
```

## 九、开发注意事项

1. **所有接口返回必须经过 Show() 函数**
2. **所有状态码必须定义到 Code.php，消息必须定义到 lang 目录**
3. **涉及金额变动的操作必须使用事务**
4. **涉及用户重复操作必须加锁 (Cache)**
5. **注释要详细，包括功能说明、参数说明、返回值说明**
6. **遵循项目规范命名**
7. **订单不存在等情况要抛出异常进行回滚**

## 十、快速开发模板

### 新增接口流程

1. **在 Model 中添加常量和方法**（如需要）
2. **在 Controller 中添加方法**
3. **在 Code.php 添加状态码**
4. **在 lang/*.php 添加多语言消息**
5. **在 route/app.php 添加路由**

### 示例：新增领取接口

```php
// 1. Model 添加方法
// app/model/CommonIncomeClaimLogModel.php

// 2. Controller 添加方法
// app/controller/product/GoodsOrderCon.php

// 3. 添加状态码
// app/helper/Code.php

// 4. 添加多语言消息
// app/lang/zh-cn.php, en.php, fr.php

// 5. 添加路由
// route/app.php
```
